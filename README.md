# 智能书签管理器

使用 AI 大模型自动对 Chrome 书签进行智能分类管理的浏览器插件。

## 功能特性

- **AI 智能分类**：配置大模型 API 后，一键对书签进行自动分类
- **自动创建文件夹**：根据分类结果在「其他书签」下创建「AI分类书签」及子文件夹
- **落地页展示**：新标签页展示全部书签，支持搜索、筛选、网格/列表视图
- **多 API 支持**：兼容 OpenAI、DeepSeek、通义千问等 OpenAI 格式 API

---

## 安装步骤

### 1. 加载扩展

1. 打开 Chrome 浏览器
2. 地址栏输入 `chrome://extensions/` 并回车
3. 打开右上角 **「开发者模式」** 开关
4. 点击 **「加载已解压的扩展程序」**
5. 选择本项目目录 `chrome-url-analysis`（包含 `manifest.json` 的文件夹）
6. 加载成功后，扩展栏会出现「智能书签管理器」图标

### 2. 固定到工具栏（可选）

点击扩展栏的拼图图标，找到「智能书签管理器」，点击图钉固定到工具栏，方便后续使用。

## 配置说明

### 首次使用前必须配置 API

1. 点击扩展图标，在弹出面板中点击 **「配置 AI 设置」**
2. 或右键扩展图标 → **「选项」**
3. 填写以下信息：

| 配置项 | 说明 | 示例 |
|--------|------|------|
| API Base URL | 大模型 API 地址 | `https://api.openai.com/v1` |
| API Key | 你的 API 密钥 | `sk-xxxxxxxx` |
| 模型名称 | 使用的模型 | `gpt-4o-mini`、`deepseek-chat` |
| 分类数量上限 | 最多创建多少个分类 | `10` |
| 根文件夹名称 | 分类书签的父文件夹名 | `AI分类书签` |

### 常用 API 配置示例

**OpenAI**
- Base URL: `https://api.openai.com/v1`
- 模型: `gpt-4o-mini`、`gpt-4o`

**DeepSeek**
- Base URL: `https://api.deepseek.com/v1`
- 模型: `deepseek-chat`

**通义千问**
- Base URL: `https://dashscope.aliyuncs.com/compatible-mode/v1`
- 模型: `qwen-turbo`、`qwen-plus`

4. 点击 **「测试连接」** 验证配置是否正确
5. 点击 **「保存设置」**

## 使用方法

### 方式一：Popup 快捷操作

1. 点击扩展图标打开 Popup
2. 查看书签统计（全部 / 已分类 / 待分类）
3. 勾选 **「对全部书签重新分类」** 可对所有书签重新归类（默认只处理书签栏根目录下的未分类书签）
4. 点击 **「AI 智能分类」** 开始分类
5. 等待进度条完成，分类结果会写入「其他书签」下的「AI分类书签」文件夹
6. 点击 **「查看所有书签」** 打开落地页

### 方式二：落地页（新标签页）

1. 打开新标签页（`Ctrl+T` 或 `Cmd+T`）
2. 默认会显示本插件的落地页，展示全部书签
3. **搜索**：在顶部搜索框输入关键词，支持按标题和 URL 搜索
4. **快捷键**：`Ctrl+K`（Windows）或 `Cmd+K`（Mac）快速聚焦搜索
5. **筛选**：点击「全部」旁的分类标签，按文件夹筛选
6. **视图切换**：点击右上角网格/列表图标切换展示方式
7. **设置**：点击齿轮图标进入配置页

## 测试指南

### 1. 基础功能测试

| 步骤 | 操作 | 预期结果 |
|-----|------|----------|
| 1 | 打开 `chrome://extensions/`，加载扩展 | 扩展列表中出现「智能书签管理器」 |
| 2 | 点击扩展图标 | 弹出 Popup，显示书签统计 |
| 3 | 打开新标签页 | 显示落地页，展示书签列表 |
| 4 | 点击「配置 AI 设置」 | 打开设置页 |

### 2. API 配置测试

| 步骤 | 操作 | 预期结果 |
|-----|------|----------|
| 1 | 在设置页填写正确的 API 信息 | 表单可正常输入 |
| 2 | 点击「测试连接」 | 显示「连接成功！」 |
| 3 | 点击「保存设置」 | 显示「设置已保存」 |
| 4 | 关闭设置页后重新打开 | 配置被正确保留 |

### 3. 智能分类测试

**建议先用少量书签测试：**

1. 在书签栏根目录添加 3–5 个不同主题的书签（如：GitHub、新闻、购物等）
2. 在 Popup 中点击「AI 智能分类」
3. 观察进度条和提示信息
4. 完成后打开书签管理器（`Ctrl+Shift+O`），在「其他书签」下查看「AI分类书签」文件夹
5. 确认书签已被移动到对应子文件夹

### 4. 落地页测试

| 步骤 | 操作 | 预期结果 |
|-----|------|----------|
| 1 | 在搜索框输入关键词 | 书签列表实时过滤 |
| 2 | 点击分类标签 | 只显示该分类下的书签 |
| 3 | 点击书签卡片 | 在新标签页打开对应网址 |
| 4 | 点击视图切换图标 | 网格/列表布局切换 |

### 5. 常见问题排查

| 现象 | 可能原因 | 处理方式 |
|------|----------|----------|
| 测试连接失败 | API Key 错误或网络问题 | 检查 Key 是否正确、是否有访问权限 |
| 分类失败 | 模型不支持 JSON 输出 | 尝试更换模型（如 `gpt-4o-mini`） |
| 落地页空白 | 书签为空或加载失败 | 刷新新标签页，检查控制台错误 |
| 书签未移动 | 权限不足 | 确认扩展已授予书签权限 |

## 项目结构

```
chrome-url-analysis/
├── manifest.json          # 扩展配置
├── icons/                 # 扩展图标
├── newtab/                # 新标签页落地页
├── popup/                 # 点击图标弹出的面板
├── options/               # 设置页
└── background/            # 后台服务（书签操作、API 调用）
```

## 隐私说明

- API Key 仅存储在浏览器本地（Chrome 同步存储），不会上传到任何第三方服务器
- 书签数据仅在本地处理，分类请求会发送到你配置的 API 服务商
